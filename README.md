# velay

一个 Windows 下的语音音效播放器(类 [Soundpad](https://enterprise.leppsoft.com/) 的思路),
用 Rust 写。按下热键把指定音频"播"进游戏语音频道(如 CS),平时自动使用真实麦克风,
播放音效时自动切到虚拟麦克风,播放结束自动切回。

详细的技术决策、分阶段计划和风险预案见 [soundpad_rust_action_guide.md](soundpad_rust_action_guide.md)。

## 原理

- 依赖 [VB-Cable](https://vb-audio.com/Cable/) 提供虚拟麦克风设备,不自研音频驱动。
- 不做"混音同播",而是在需要播放音效时把系统默认录音设备切换成虚拟麦克风,
  播放结束后再切回真实麦克风。
- 设备切换通过 Windows 非公开 COM 接口 `IPolicyConfig::SetDefaultEndpoint` 实现——
  这是业内切换默认音频设备的通行做法(EarTrumpet、AudioSwitcher 等工具同款思路),
  不注入游戏进程、不读写游戏内存。

## 环境要求

- Windows 10 / 11
- 已安装 [VB-Cable](https://vb-audio.com/Cable/)(免费,捐赠软件模式)
- Rust stable 工具链

## 使用

### 图形界面(GUI,推荐)

直接**双击 `soundpad.exe`**(或开发期 `cargo run`,不带任何子命令)即可打开图形界面:

- **音效列表**:点「＋ 添加音频文件」选文件(支持 mp3 / wav / flac / ogg),或把音频
  文件**直接拖入窗口**。每行可播放(▶ / 双击)、单独绑键(高级)、改名、删除。
  名称默认取文件名(已清洗特殊字符)。
- **播放/暂停/停止**:底部按钮,或用**全局热键**(默认 `F8` = 播放/暂停,`F9` = 停止)。
  交互模型是「在列表里选中一个音效 → 回游戏按播放/暂停热键触发」,**不用背多个键位**。
- **暂停语义**:暂停时会自动把默认录音设备**切回真麦克风**(这样暂停期间说话队友能
  听见),恢复播放时再切到虚拟声卡。
- **音量**:底部滑条,实时生效。
- **热键**:设置区「全局热键」可把播放/暂停、停止**改绑到键盘任意键**(点「录制」后按
  目标键,`Esc` 取消);「恢复默认热键」一键还原。若目标键已被占用会**提示并阻止绑定**。
- **播放时自动开麦(高级,默认关)**:开启后播放时用 `SendInput` 模拟按住游戏语音键,
  松开自动停。属于「宏」范畴,**是否有效需在游戏内实测**;游戏用 Raw Input 过滤合成
  输入时可能无效,此时请改用游戏内「语音激活」模式。
- **切换范围(路由模式,阶段七)**:设置区单选「全局」或「仅选中程序」。
  - **全局**(默认):切换**系统默认**麦克风,所有「跟随默认」的程序(含微信语音等)
    播放期间都会收到音效——旧行为。
  - **仅选中程序**:只切换目标程序(如 `cs2.exe`)的麦克风,系统默认不动,其它程序
    完全不受影响。目标程序默认从「有窗口的程序」列表里选(好认),也可勾选「显示
    全部进程」或输入关键词筛选;游戏启动后点「🔄 刷新」再选。
  - 该区还有「**⚠ 紧急恢复**」按钮:若程序被强杀导致目标程序卡在虚拟麦克风,点它
    清除**所有**应用的 per-app 音频设备覆盖(注意会连带清掉你在 Windows 设置里
    手动设过的项)。
- **设备**:可手动指定真麦克风、虚拟声卡录音端(默认自动识别 VB-Cable)。

所有设置实时写入 exe 同目录的 `config.json`。目前 GUI 与旧命令行子命令共存。

### 运行方式

开发期(没有单独编译)在项目根目录用 `cargo run --` 加子命令,`--` 后面的内容
会原样传给程序:

```
cargo run -- daemon
cargo run -- play C:\sounds\hello.mp3
```

发布后(`cargo build --release` 产出的 `soundpad.exe`)直接把子命令跟在 exe
后面即可,两种方式命令行为完全一致:

```
soundpad.exe daemon
soundpad.exe play C:\sounds\hello.mp3
```

下文统一用 `soundpad <子命令>` 指代,开发期请自行替换成 `cargo run -- <子命令>`。

### 命令列表

```
soundpad                无参数 → 启动图形界面(GUI,推荐)
soundpad gui            显式启动图形界面
soundpad daemon         启动热键守护进程(旧 per-key 模式,游戏内按热键触发音效)
soundpad play <文件>    播放单个音频文件(自动切换录音设备,阻塞到播完/按 Enter)
soundpad devices        列出所有输出设备(找 CABLE Input 用)
soundpad rec            列出所有录音设备(找真麦克风/CABLE Output 用)
soundpad <关键词>       手动切换默认录音设备(不区分大小写、子串匹配,调试用)
```

诊断用子命令(排查 per-app 路由 / 控制器问题时用):

```
soundpad apps                  列出有窗口的程序(确认目标程序的 exe 名)
soundpad procs [关键词]        列出全部运行进程名(目标程序没有窗口时兜底)
soundpad approute-selftest     自测 per-app 路由接口(只对本进程设置再清除,无副作用)
soundpad approute-diag <进程>  对指定进程逐 PID / 逐角色尝试切换,逐条打印结果
soundpad controller-test       走 GUI 同款控制器路径试播一次(验证切换+播放+监听)
```

### 调整设置(config.json)

所有设置都存在 exe 同目录的 `config.json` 里,没有独立的设置命令,直接编辑
这个文件后重新运行 `soundpad daemon` 生效:

| 字段 | 说明 |
|---|---|
| `hotkeys` | 热键 → 音频文件的映射,见下方"热键守护进程"一节 |
| `real_mic_device_id` | 真实麦克风的设备 ID,首次播放会自动写入;需要改回其他麦克风时,手动运行 `soundpad <新麦克风关键词>` 切换后再触发一次播放即可自动更新 |
| `real_mic_device_name` | 对应上一字段的友好名称,仅用于日志显示,无需手动改 |

文件不存在或字段缺失时程序会用默认值(空),不会报错;`hotkeys` 为空时
`soundpad daemon` 会打印配置示例并退出,提示先编辑。

### 热键守护进程(daemon)

> 这是阶段四的旧 per-key 模式,与 GUI 共存但功能较少:**只支持「全局」路由**
> (不吃 `routing_mode` / `target_apps`),也不支持本地监听和音量设置——这些都是
> GUI 控制器的功能。日常使用推荐直接用 GUI。

首次运行会提示 `config.json` 中缺少 `hotkeys` 配置。在 exe 同目录的 `config.json`
里添加热键 → 音频文件的映射,键名支持 `NumpadN`(小键盘 0-9)和 `F1`-`F12`,
其中一个键的值固定写 `__STOP__` 作为停止键:

```json
{
  "hotkeys": {
    "Numpad1": "C:\\sounds\\hello.mp3",
    "Numpad2": "C:\\sounds\\laugh.mp3",
    "Numpad0": "__STOP__"
  }
}
```

保存后重新运行 `soundpad daemon`。真实麦克风的设备 ID 会在首次播放时自动记录进
`config.json` 的 `real_mic_device_id` 字段,之后每次播放都会自动切回。

### 退出与异常恢复

正常退出(关闭 GUI 窗口 / 关闭 daemon 窗口 / Ctrl+C)时,程序会依次:松开自动开麦的
语音键 → 清除目标程序的 per-app 覆盖 → 把系统默认录音设备切回真麦克风。GUI 关窗时会
**等控制器确认恢复完成**(最多 3 秒)再退出,不会出现"进程先没了、设备没恢复"的竞态。

被任务管理器强杀时退出钩子不触发,三类残留的自动恢复能力如下:

| 残留 | 后果 | 自动恢复 |
|---|---|---|
| 系统默认卡在 CABLE Output(全局模式) | 所有程序都听不到你说话 | 下次运行 `soundpad`(任意命令)时自动检测并切回 |
| 目标程序卡在虚拟麦克风(per-app 模式) | 只有该程序听不到你说话 | 下次启动时自动清理;**若该程序当时没在运行**,程序会记住并在它启动后自动清除(见下) |
| 自动开麦的语音键没松开 | 游戏内麦克风一直开着 | **无法自动恢复**(按键状态在游戏那侧),重开游戏或手动按一下该键即可 |

per-app 覆盖是**持久化**的(按程序身份保存、重启不消失),而清除它需要目标程序**正在
运行**(接口靠 PID 定位应用)。因此程序会在 exe 同目录维护一个 `per_app_pending.json`
标记文件:设覆盖前写入、恢复成功后删除。若启动时发现标记还在但目标程序没运行,会转入
后台守望,等目标程序一启动就自动清除覆盖——期间你不需要做任何事。

实在需要立即清除时,可点设置区的「⚠ 紧急恢复」(注意它会清掉**所有**应用的 per-app
音频设备设置,包括你在 Windows 设置里手动设过的)。

## 已知风险

- `IPolicyConfig` 是非公开接口,不同 Windows 版本理论上存在 ABI 差异,如遇
  `E_NOINTERFACE` 需要重新核对 GUID / vtable 布局(现已在切换失败时打印错误,
  不会再静默失败)。
- 强制结束进程(任务管理器杀进程等)不会触发退出钩子,麦克风可能暂时卡在
  CABLE Output,需下次启动或手动切回,见上文"退出与异常恢复"。
- 「全局」模式下,播放音效期间所有「跟随系统默认录音设备」的程序(不止 CS,也包括
  微信语音等)都会暂时收到音乐而非你的声音;播完/暂停即恢复。**改用「仅选中程序」
  模式可彻底避免牵连其它程序**。
- **前提(重要)**:游戏的麦克风选择有三层优先级 —— `CS2 内指定的麦克风` >
  `Steam 语音设置` > `Windows 默认录音设备`。只要 CS2 或 Steam 里锁定了具体麦克风,
  本工具的切换对它就无效。请把 **CS2 和 Steam 的语音输入设备都设为「默认」**。
- 「仅选中程序」模式依赖未公开接口 `IAudioPolicyConfig`,其 per-app 设置是**持久化**的
  (重启不消失),且清除它要求目标程序正在运行。强杀本工具会留下残留——已有自动恢复
  机制兜底,详见上文「退出与异常恢复」。
- **「播放时自动开麦」被强杀时无法兜底**:合成按键的"按下"状态在游戏进程里,本工具
  一旦没来得及发出松键就被强杀,游戏会一直以为你按着语音键(麦克风常开),且下次启动
  也无从知晓。正常退出路径已做进程级兜底;介意这一点就别开这个选项。
- 更多风险与预案见 [soundpad_rust_action_guide.md](soundpad_rust_action_guide.md#11-关键风险总览按优先级)。
